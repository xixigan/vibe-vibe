# 11.2 云端仓库与认证配置

## 选择代码托管平台

| 平台 | 特点 | 适用场景 |
|------|------|---------|
| GitHub | 国际主流，生态丰富 | 海外项目、开源项目 |
| Gitee | 国内访问快 | 国内项目、团队协作 |

## 创建云端仓库

告诉 AI："帮我在 GitHub 上创建一个名为 xxx 的仓库，并关联到本地项目"

AI 会帮你完成：
1. 在 GitHub 网页创建仓库
2. 在本地添加远程仓库地址
3. 推送初始代码

## 认证方式选择

**推荐：SSH 密钥认证**
- 一次配置，长期免密
- 安全性高

**备选：HTTPS + Token**
- 适合临时使用
- 需要定期更新 Token

## SSH 配置步骤

告诉 AI："配置 SSH 密钥连接到 GitHub"

AI 会帮你：
1. 生成 SSH 密钥对（如果还没有）
2. 显示公钥内容
3. 指导你在 GitHub 设置中添加公钥
4. 测试连接是否成功

## 常见配置问题

**推送失败**
- 检查网络连接
- 确认有仓库写入权限
- 确认 SSH 密钥已添加

**权限 denied**
- 检查 SSH 代理是否运行
- 确认使用了正确的密钥
